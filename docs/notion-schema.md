# Notion Schema

Create these databases manually first. After the Notion integration token is available, the same schema can be mapped to API writes.

## Database: `Upwork Bidding Pipeline`

Purpose: track job candidates, proposal drafts, human review, submitted bids, and outcomes.

| Property | Type | Required | Notes |
| --- | --- | --- | --- |
| Job Title | Title | Yes | Primary title. |
| Upwork URL | URL | No | Use only manually provided/authorized URL. |
| Source | Select | Yes | `manual_paste`, `csv_sample`, `authorized_feed`, `other`. |
| Status | Status | Yes | See status values below. |
| Fit Score | Number | Yes | 0-100. |
| Risk Flags | Multi-select | No | See risk flags below. |
| Budget Type | Select | No | `fixed`, `hourly`, `unknown`. |
| Budget | Text | No | Human-readable budget range. |
| Payment Method Verified | Select | Yes | `verified`, `unverified`, `unknown`. |
| Client Country | Text | No | Country if provided. |
| Required Skills | Multi-select | No | Skills/keywords from job. |
| Proposal Draft | Rich text | No | Draft generated by `proposal-drafter`. |
| Portfolio Picks | Relation or Multi-select | No | Prefer relation to `Portfolio Catalog` later. |
| Reviewer | Person or Text | No | Human reviewer. |
| Submitted At | Date | No | Only set after manual bid submit. |
| Connects Used | Number | No | Only set after manual bid submit. |
| Outcome | Select | No | `none`, `follow_up`, `won`, `lost`, `archived`. |
| Notes | Rich text | No | Human notes. |
| Duplicate Key | Text | Yes | Stable dedupe key. |

Status values:

- `New`
- `Need Review`
- `Approved to Bid`
- `Bid Sent`
- `Rejected`
- `Won`
- `Lost`
- `Follow Up`

Risk flags:

- `unclear_scope`
- `low_budget`
- `unrealistic_timeline`
- `requires_free_work`
- `requires_off_platform_contact`
- `suspicious_payment`
- `payment_unverified`
- `credential_request`
- `policy_risk`
- `not_illiyin_fit`

Default filter:

- Fixed price jobs must be at least USD 500.
- Hourly jobs must be at least USD 15/hour.
- Payment method must be verified.
- If a job does not pass the default filter, keep it as `Rejected` or `Need Review`.
- Sales lead can manually override this for a specific case by adding the reason in `Notes`.

Recommended views:

- `Inbox`: Status is `New` or `Need Review`.
- `Ready to Bid`: Status is `Approved to Bid`.
- `Submitted`: Status is `Bid Sent` or `Follow Up`.
- `Closed`: Status is `Rejected`, `Won`, or `Lost`.
- `High Fit`: Fit Score >= 80.

Duplicate key formula recommendation:

```text
source_url if present, else normalized(title + client_country + budget)
```

For the first manual version, store the duplicate key as text. Later, compute it in code before writing to Notion.

## Database: `Portfolio Catalog`

Purpose: source of truth for proposal portfolio selection.

| Property | Type | Required | Notes |
| --- | --- | --- | --- |
| Project Name | Title | Yes | Portfolio/case study name. |
| Portfolio ID | Text | Yes | Stable ID, e.g. `portfolio-crm-automation`. |
| Category | Select | Yes | `automation`, `web_app`, `ai_integration`, `crm`, `data`, `mobile`, `other`. |
| Tech Stack | Multi-select | No | Tools/frameworks used. |
| Proof URL | URL | No | Public or shareable proof. |
| Case Study Summary | Rich text | Yes | Short truthful summary. |
| Best For Keywords | Multi-select | No | Keywords for matching jobs. |
| Status | Select | Yes | `active`, `needs_update`, `private`, `archived`. |

Recommended views:

- `Active Portfolio`: Status is `active`.
- `Needs Update`: Status is `needs_update`.
- `By Category`: grouped by Category.

## Notion API Mapping

Environment variables:

```text
NOTION_TOKEN=
NOTION_UPWORK_PIPELINE_DATABASE_ID=
NOTION_PORTFOLIO_CATALOG_DATABASE_ID=
```

Write rules:

- Create new page only when `Duplicate Key` does not exist.
- Update existing page when `Duplicate Key` already exists.
- Do not set `Bid Sent` unless human confirmation includes `bid_sent`.
- Store generated JSON in page body or `Notes` when debugging is needed.

Minimal create payload fields:

```json
{
  "Job Title": "Build CRM automation in Airtable",
  "Source": "manual_paste",
  "Status": "Need Review",
  "Fit Score": 82,
  "Risk Flags": ["unclear_scope"],
  "Budget Type": "fixed",
  "Budget": "$500-$1500",
  "Payment Method Verified": "verified",
  "Client Country": "United States",
  "Required Skills": ["automation", "airtable", "zapier"],
  "Duplicate Key": "manual_paste:build-crm-automation-in-airtable:united-states"
}
```
